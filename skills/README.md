# Skills

Add each skill in its own folder here:

```text
skills/
  my-skill/
    SKILL.md
```

The `npx skills` CLI discovers `SKILL.md` files in this directory automatically.


## `/new-pr`

Assists in creating a new pull request, guiding the agent to look for a PR template and to ask which branch the PR should merge into.

Example usage:

```
# Basic
/new-pr

# Include a target branch and a reference:
/new-pr into main. Include a reference to the original task: [Optimize the thingy](https://app.asana.com/xxxxxxx)
```


# Working with a "Ralph" loop

Sometimes it can be beneficial to break down larger efforts into discrete tasks that get worked on separately. This is particularly useful when trying to reduce context bloat for larger efforts. The skills below provide a simple workflow for progressing from a detailed implementation plan into work completed via multiple sub-tasks. This workflow requires the files below, which the skills can create for you. These files are not intended to be committed to source control as they will be rewritten or altered as needed for the work being done or the user's working preferences.

| File | Purpose |
| --- | --- |
| `.ralph/todo.md` | The task list. Tracks the source plan file, the unchecked tasks to work through, and a Completed section. Generated by `/task-to-ralph` from a spec, then updated by `/ralph-work` as tasks are completed. |
| `.ralph/prompt.md` | The prompt dispatched to a fresh subagent for each task iteration. Tells the subagent how to pick a task from `todo.md`, complete it (build, test, commit), and update the todo list. Scaffolded from the `ralph-work` skill's bundled template on first run. |


## `/task-to-ralph`

Breaks a spec file into discrete, committable tasks and writes them to `.ralph/todo.md` for use with `/ralph-work`.

Example usage:

```
# Ask which spec to use:
/task-to-ralph

# Provide the spec file directly:
/task-to-ralph specs/my-feature/plan.md
```


## `/ralph-work`

Works through the tasks in `.ralph/todo.md` one at a time, dispatching a fresh subagent per task so context stays clean between tasks. On first run, offers to scaffold `.ralph/prompt.md` and add `.ralph/` to `.gitignore`.

Example usage:

```
/ralph-work
```
